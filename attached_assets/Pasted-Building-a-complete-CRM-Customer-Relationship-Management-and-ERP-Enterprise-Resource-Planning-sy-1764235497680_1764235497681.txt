API Gateway (Frontend to Backend)Purpose: The single entry point for all frontend client requests. It routes requests to the appropriate microservice, handles authentication, and performs rate limiting.Technology: Can be built using a lightweight Python framework (like FastAPI or Flask) or a dedicated service (Nginx/Kong). The gateway should translate the public URL (/api/v1/leads) to the internal service URL (http://sales-service:8001/leads).
